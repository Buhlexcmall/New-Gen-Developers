## Agenda

* Compare architectural styles

## Discussion Summary

* The team discussed possible architectural styles including Monolithic, Microservices, and Layered Architecture.
* Members agreed that Layered Architecture would best suit the Student Productivity App because it is easier to manage, test, and maintain.
* The team reviewed the need for clear separation between the frontend, backend, and database layers.
* System modules such as User Management, Schedule Management, Assignment Tracking, Notifications, and Dashboard Management were identified.
* The team also discussed creating UML Component Diagrams and Deployment Diagrams for the architecture documentation.

## Decisions Made

* Layered Architecture was selected as the system architecture.
* The frontend, backend, and database would be separated into different layers.
* Draw.io would be used to create architecture diagrams.
* GitHub Issues would be updated to track architecture tasks.
